## Patterns for Playbook

**Pre-destroy validation:**
1. Check for orphan private endpoints (esp. from manual testing).
2. Check for orphan Container Apps Environments and stale service association links.
3. Delete orphans before 	erraform destroy to avoid subnet blocking.

**Post-destroy:**
1. Wait ~5 min for stale SAL propagation if Container Apps were involved.
2. If destroy fails on subnets, check Azure portal for stale service associations before retry.

**Timing notes:**
- Foundry destroy: ~10 min (clean)
- Networking destroy: ~45 min (30 min state refresh + 15 min Azure destroy)
